R1 — Object Movement Sequence
The robot shall execute the MoveObject action followed by the WaitObject action in sequence.
If MoveObject fails, the sequence shall stop and the robot shall move to the next fallback branch.
If MoveObject or WaitObject is still running, the system shall wait for completion.

    <Action ID="WaitObject" wait_duration="3"/>

R2 — MoveObject Behavior
The MoveObject action shall perform a movement operation toward the commanded pose defined in action, using inputs:

{move_base_finished} – indicator for completion,
{move_base_idle} – motion planner idle state,
{pose_queue_size} – queue management parameter. Result Behavior: It shall return SUCCESS if the object reached its target position successfully.

R3 — WaitObject Behavior
After a successful object movement, the robot shall execute the WaitObject action to remain idle for 3 seconds before continuing.
Code Reference:

<Action ID="WaitObject" wait_duration="3"/>
R4 — Object Return Sequence
If the MoveObject sequence fails, the robot shall execute the ReturnObject sequence which includes:

Executing ReturnObject to move the object to its home pose (using {home_pose_far}, {move_base_finished}, and {move_base_idle} inputs).
Executing WaitObject to wait for 3 seconds. If ReturnObject fails, the Fallback shall proceed to the next recovery step. 
Code Reference:

    <Action ID="ReturnObject"/>
    <Action ID="WaitObject" wait_duration="3"/>
</Sequence>

R5 — ReturnObject Behavior
The ReturnObject action shall navigate the object from its current state back to the home position specified by {home_pose_far}.
It shall indicate success when the vehicle or manipulator is back at home and motion states (move_base_finished, move_base_idle) are nominal.
Code Reference:

<Action ID="ReturnObject" .../>
R6 — Standby Mode
If both MoveObject and ReturnObject sequences fail, the system shall execute the StandbyObject action to place the robot in a stable standby or idle state ready for further tasks.
Code Reference:

<Action ID="StandbyObject"/>
R7 — Hierarchical Fallback Logic
The behavior control shall abide by the Fallback pattern rules:

Attempt the MoveObject sequence first.
If it fails, execute the ReturnObject sequence.
If both fail, execute StandbyObject unconditionally. The first sequence that succeeds shall end the Fallback execution. Dictionary Reference:
Fallback — Execute [actions] in order. If [action] fails, move to the next [action] until one succeeds.
