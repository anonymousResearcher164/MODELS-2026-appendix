REQ-1:
The system shall execute the full tray handling sequence one time only.

Source: Sequence Node

REQ-2:
The system shall execute actions in the following order: LookAtTheTray, DetectHandles, PickTray, LookUp, MoveBack, GoToTable, and PlaceTray.

REQ-3:
The system shall not begin execution of any subsequent action until the current action reports a SUCCESS condition.

REQ-4:
If any action in the sequence reports FAILURE, the system shall stop executing the remaining actions in the sequence.

REQ-5:
While any action is in progress (RUNNING), the system shall wait for it to finish before proceeding.

Source: Action Node — LookAtTheTray
(Atomic action, no modifiers)

REQ-6:
The system shall direct its sensors or vision system toward the tray to establish visual focus before handle detection begins.

Source: RetryUntilSuccessful Node (num_attempts=3)
Dictionary Reference (RetryUntilSuccessful Template):
“Keep executing [action] until it returns SUCCESS. If retry returns FAILURE, restart [action]. Else return RUNNING.”

(Note: The BT XML misspells “RetryUntilSuccessful” as “RetryUntilSuccesful,” but we use the intended meaning.)

REQ-7:
The system shall attempt to execute the DetectHandles action up to 3 times or until it succeeds.

REQ-8:
If the DetectHandles action fails within 3 attempts, the RetryUntilSuccessful node shall return FAILURE, causing the sequence to stop.

Source: Timeout Node (msec=300)
Dictionary Reference (Timeout Template):
“If the execution of an [action] does not end within a specified duration, interrupt it and return a specific result.”

REQ-9:
The system shall limit the execution time of the DetectHandles action to a maximum of 300 milliseconds per attempt.

REQ-10:
If DetectHandles does not complete within 300 milliseconds, the Timeout node shall interrupt DetectHandles and report FAILURE for that attempt.

Source: Action Node — DetectHandles
(Atomic action executed under Timeout and Retry control)

REQ-11:
The system shall use visual or sensor data to identify the positions of tray handles during each attempt of DetectHandles.

Source: Action Node — PickTray
(Action executed after handle detection succeeds)

REQ-12:
After successful handle detection, the system shall grasp the tray using appropriate manipulators.

Source: Action Node — LookUp
(Atomic action)

REQ-13:
After picking up the tray, the system shall tilt or move its vision system upward to reorient its field of view.

Source: Action Node — MoveBack
(Atomic action)

REQ-14:
The system shall move backward from its current position after lifting the tray to create clearance from the pickup location.

Source: Action Node — GoToTable
(Atomic action)

REQ-15:
The system shall move toward the designated table area while securely holding the tray.

Source: Action Node — PlaceTray
(Final atomic action)

REQ-16:
The system shall release or place the tray on the table in a stable orientation completing one operational cycle.
